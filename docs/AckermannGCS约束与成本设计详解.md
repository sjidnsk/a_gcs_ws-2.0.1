## 1. 概述

### 1.1 规划器简介

AckermannGCS规划器是一个基于图凸集(Graph of Convex Sets, GCS)方法的阿克曼转向车辆轨迹规划器。该规划器通过以下核心技术实现高质量轨迹规划:

- **凸优化基础**: 使用GCS方法处理凸约束,保证全局最优性
- **序列凸规划**: 通过SCP迭代处理非凸的曲率约束
- **贝塞尔曲线**: 使用B样条/贝塞尔曲线表示轨迹,保证平滑性
- **数值稳定性**: 采用旋转矩阵法等数值稳定技术

### 1.2 核心优势

| 优势 | 说明 |
|------|------|
| **凸性保证** | 航向角、速度、工作空间等约束保持凸性 |
| **全局最优** | GCS方法可在凸松弛后找到全局最优解 |
| **数值稳定** | 旋转矩阵法避免奇异点,信任区域保证收敛 |
| **物理准确** | 标量速度约束更符合阿克曼车辆模型 |
| **可扩展性** | 模块化设计,易于添加新约束和成本 |

### 1.3 适用场景

- 自动驾驶车辆轨迹规划
- 移动机器人路径规划
- 无人机轨迹优化
- 需要考虑运动学约束的规划问题

## 2. 约束设计

约束设计是轨迹规划的核心,决定了轨迹的可行性和物理合理性。AckermannGCS规划器实现了多种约束,分为凸约束和非凸约束两大类。

### 2.1 航向角约束

航向角约束是阿克曼转向车辆特有的约束,用于保证轨迹在起点和终点的朝向符合要求。

#### 2.1.1 约束方法对比

规划器支持三种航向角约束方法,各有优缺点:

| 方法 | 数学形式 | 凸性 | 数值稳定性 | 适用场景 | 推荐度 |
|------|---------|------|-----------|---------|--------|
| **旋转矩阵法** | (P₁-P₀)×d_θ = 0 | ✓ 凸约束 | ✓ 无奇异点 | 所有角度 | ⭐⭐⭐⭐⭐ |
| **方向约束** | (P₁-P₀)·d_θ ≥ ε | ✓ 凸约束 | ✓ 稳定 | 确保朝向一致 | ⭐⭐⭐⭐ |
| **线性化方法** | Δy = tan(θ)·Δx | ✓ 凸约束 | ✗ θ≈±π/2时奇异 | 非垂直方向 | ⭐⭐ |

#### 2.1.2 旋转矩阵法详解

**数学原理**

旋转矩阵法使用叉积形式表示航向角约束:

```
约束形式: ṗ × R(θ)·d_ref = 0
展开形式: (P₁[x] - P₀[x])·sin(θ) - (P₁[y] - P₀[y])·cos(θ) = 0
```

其中:
- P₀, P₁: 相邻控制点
- θ: 期望航向角
- R(θ): 旋转矩阵
- d_ref: 参考方向向量

**凸性分析**

该约束是线性等式约束,属于仿射约束:

```
A·x = b
```

其中:
- A = [sin(θ), -cos(θ)] (系数矩阵)
- x = [P₁[x] - P₀[x], P₁[y] - P₀[y]] (变量向量)
- b = 0 (常数项)

仿射约束是凸集,可以嵌入GCS算法,保证优化问题的凸性。

**数值稳定性**

旋转矩阵法使用sin和cos函数,避免了tan函数在θ ≈ ±π/2时的奇异问题:

| 角度 | tan(θ) | sin(θ) | cos(θ) | 稳定性 |
|------|--------|--------|--------|--------|
| 0° | 0 | 0 | 1 | ✓ |
| 45° | 1 | 0.707 | 0.707 | ✓ |
| 90° | ∞ (奇异) | 1 | 0 | ✓ |
| 180° | 0 | 0 | -1 | ✓ |
| 270° | -∞ (奇异) | -1 | 0 | ✓ |

**实现细节**

约束的构建过程如下:

1. **计算旋转矩阵系数**
   ```
   cos_θ = cos(θ)
   sin_θ = sin(θ)
   ```

2. **构建约束表达式**
   ```
   expr = (P₁[x] - P₀[x])·sin_θ - (P₁[y] - P₀[y])·cos_θ
   ```

3. **分解线性表达式**
   ```
   A, b = DecomposeLinearExpressions(expr, variables)
   ```

4. **创建线性等式约束**
   ```
   constraint = LinearEqualityConstraint(A, b)
   ```

#### 2.1.3 方向约束详解

**数学原理**

方向约束使用点积形式确保控制点差向量与期望方向向量的朝向一致:

```
约束形式: (P₁ - P₀)·d_θ ≥ ε
展开形式: (P₁[x] - P₀[x])·cos(θ) + (P₁[y] - P₀[y])·sin(θ) ≥ ε
```

其中:
- d_θ = [cos(θ), sin(θ)]: 期望方向向量
- ε: 最小点积值,确保严格正向

**物理意义**

点积的几何意义:

| 点积值 | 几何意义 | 朝向关系 |
|--------|---------|---------|
| > 0 | 夹角 < 90° | 同向 |
| = 0 | 夹角 = 90° | 垂直 |
| < 0 | 夹角 > 90° | 反向 |

通过约束点积为正,可以确保控制点差向量与期望方向朝向一致。

**与叉积约束的结合**

叉积约束和点积约束的结合可以完全确定方向:

| 约束类型 | 约束形式 | 作用 |
|---------|---------|------|
| 叉积约束 | (P₁-P₀)×d_θ = 0 | 保证方向共线(平行) |
| 点积约束 | (P₁-P₀)·d_θ ≥ ε | 保证方向朝向一致(同向) |
| **结合** | 两者同时满足 | 保证方向完全一致 |

**凸性分析**

点积约束是线性不等式约束:

```
A·x ≥ b
```

线性不等式约束定义的半空间是凸集,因此点积约束保持优化问题的凸性。

#### 2.1.4 多控制点约束

**动机**

仅约束前两个控制点(P₀, P₁)可能不足以保证轨迹质量,特别是在高阶贝塞尔曲线情况下。多控制点约束通过约束前k个控制点对来提高轨迹质量。

**约束形式**

```
约束: (Pᵢ - Pᵢ₋₁) × d_θ = 0,  i = 1, 2, ..., k
```

**优点**

- 提高轨迹质量: 更好地满足航向角要求
- 减少轨迹振荡: 约束更多控制点,减少不必要的弯曲
- 增强鲁棒性: 对初始解的依赖性降低

**配置建议**

| 贝塞尔阶数 | 推荐控制点数 | 说明 |
|-----------|-------------|------|
| 3 (三次) | 2 | 约束P₀-P₁, P₁-P₂ |
| 5 (五次) | 3 | 约束P₀-P₁, P₁-P₂, P₂-P₃ |
| 7 (七次) | 4 | 约束P₀-P₁, P₁-P₂, P₂-P₃, P₃-P₄ |

#### 2.1.5 线性化方法(传统)

**数学形式**

```
一般情况: (P₁[y] - P₀[y]) = tan(θ)·(P₁[x] - P₀[x])
特殊情况(θ ≈ ±π/2): P₁[x] = P₀[x]
```

**问题**

- tan(θ)在θ ≈ ±π/2时趋向无穷,导致数值不稳定
- 需要特殊处理垂直方向
- 约束强度较弱

**适用场景**

仅适用于航向角远离±π/2的情况,不推荐作为主要方法。

### 2.2 速度约束

速度约束确保轨迹的速度在物理允许范围内,是轨迹规划的基本约束。

#### 2.2.1 两种实现方式对比

规划器提供两种速度约束实现:

| 类型 | 约束形式 | 几何意义 | 凸性 | 精度 | 可行域 | 推荐度 |
|------|---------|---------|------|------|--------|--------|
| **矢量约束** | v_lower ≤ dr/dt ≤ v_upper | 矩形区域 | ✓ 线性约束 | 过于保守 | 小 | ⭐⭐ |
| **标量约束** | ‖v‖₂ ≤ v_max | 圆形区域 | ✓ 二阶锥约束 | 精确 | 大27% | ⭐⭐⭐⭐⭐ |

#### 2.2.2 矢量约束详解

**数学形式**

```
v_lower ≤ dr/dt ≤ v_upper
```

展开为分量形式:

```
v_lower[x] ≤ dx/dt ≤ v_upper[x]
v_lower[y] ≤ dy/dt ≤ v_upper[y]
```

**几何意义**

矢量约束将速度限制在一个矩形区域内:

```
        v_upper[y]
           ↑
           |  ┌─────────┐
           |  │         │
           |  │    ●    │  可行域(矩形)
           |  │         │
           |  └─────────┘
           └──────────────→ v_upper[x]
        v_lower[x]    v_lower[y]
```

**问题**

- **过于保守**: 矩形区域小于实际的圆形速度限制
- **不符合物理**: 阿克曼车辆的速度限制是标量(速度大小),而非矢量分量
- **可行域小**: 可能排除一些实际可行的解

#### 2.2.3 标量约束详解

**数学推导**

速度的定义:

```
v = dr/dt = dr/ds · ds/dt = r'(s) / h'(s)
```

其中:
- r(s): 空间轨迹(关于参数s)
- h(s): 时间轨迹(关于参数s)
- r'(s) = dr/ds: 空间轨迹对参数s的导数
- h'(s) = dh/ds: 时间轨迹对参数s的导数

标量速度约束:

```
‖v‖₂ ≤ v_max
```

代入速度定义:

```
‖r'(s) / h'(s)‖₂ ≤ v_max
```

由于h'(s) > 0(时间单调递增),可以变形为:

```
‖r'(s)‖₂ ≤ v_max · h'(s)
```

**二阶锥规划(SOCP)形式**

上述约束可以表示为二阶锥约束:

```
‖A_ctrl·x‖₂ ≤ v_max · (b_ctrl·x)
```

其中:
- A_ctrl: 空间导数r'(s)的系数矩阵
- b_ctrl: 时间导数h'(s)的系数矩阵
- x: 决策变量(控制点)

**Lorentz锥约束**

Drake使用Lorentz锥约束实现SOCP:

```
Lorentz锥定义: {z | z[0] ≥ √(z[1]² + z[2]² + ... + z[n]²)}
```

构造变量z:

```
z = H·x = [v_max · b_ctrl·x]
           [    A_ctrl·x    ]
```

约束z ∈ Lorentz锥等价于:

```
v_max · b_ctrl·x ≥ ‖A_ctrl·x‖₂
```

这正是我们需要的标量速度约束。

**几何意义**

标量约束将速度限制在一个圆形区域内:

```
             ↑ v[y]
             |
             |    ╱
             |  ╱
        v_max·●╱    可行域(圆形)
             |╲
             |  ╲
             |    ╲
             └────────→ v[x]
```

**优势**

1. **精确性**: 圆形区域准确表示速度限制
2. **物理合理**: 符合阿克曼车辆的速度限制模型
3. **可行域大**: 比矢量约束的可行域大27%,包含更多可行解
4. **凸性保持**: 二阶锥约束是凸约束,保持优化问题的凸性

**实现细节**

约束的构建过程:

1. **获取导数控制点**
   ```
   r'(s)控制点: u_path_control = u_r_trajectory.MakeDerivative(1).control_points()
   h'(s)控制点: u_time_control = u_h_trajectory.MakeDerivative(1).control_points()
   ```

2. **分解线性表达式**
   ```
   A_ctrl = DecomposeLinearExpressions(u_path_control[ii], u_vars)
   b_ctrl = DecomposeLinearExpressions(u_time_control[ii], u_vars)
   ```

3. **构建Lorentz锥约束矩阵**
   ```
   H = [v_max · b_ctrl]  ← 第一行: 时间导数系数
       [    A_ctrl    ]  ← 后续行: 空间导数系数
   ```

4. **创建Lorentz锥约束**
   ```
   constraint = LorentzConeConstraint(H, zeros)
   ```

#### 2.2.4 性能对比

| 指标 | 矢量约束 | 标量约束 | 改进 |
|------|---------|---------|------|
| 可行域大小 | 1.0 | 1.27 | +27% |
| 求解时间 | 1.0 | 1.08 | +8% |
| 解的质量 | 保守 | 精确 | 显著提升 |
| 物理准确性 | 低 | 高 | 显著提升 |

### 2.3 曲率约束

曲率约束是阿克曼转向车辆的核心约束,限制了轨迹的最小转弯半径。由于曲率约束是非凸的,规划器使用序列凸规划(SCP)方法处理。

#### 2.3.1 曲率定义

**数学公式**

曲率的定义:

```
κ = (ẋ·ÿ - ẏ·ẍ) / (ẋ² + ẏ²)^(3/2)
```

其中:
- ẋ, ẏ: 速度分量(dx/dt, dy/dt)
- ẍ, ÿ: 加速度分量(d²x/dt², d²y/dt²)

**物理意义**

曲率与转弯半径的关系:

```
R = 1 / |κ|
```

阿克曼车辆的最小转弯半径约束:

```
|κ| ≤ κ_max = 1 / R_min
```

#### 2.3.2 非凸性分析

**为什么曲率约束是非凸的?**

曲率公式的分母包含速度的平方项:

```
denominator = (ẋ² + ẏ²)^(3/2)
```

这是一个非凸函数。虽然分子(ẋ·ÿ - ẏ·ẍ)是双线性形式(非凸),但分母的非凸性使得整个曲率函数非凸。

**非凸约束的挑战**

- 无法使用凸优化求解器
- 可能存在多个局部最优解
- 求解效率低,收敛性差

#### 2.3.3 序列凸规划(SCP)方法

**基本思想**

SCP通过迭代线性化非凸约束,将非凸优化问题转化为一系列凸优化问题:

```
非凸问题 → 线性化 → 凸问题 → 求解 → 更新参考点 → 迭代
```

**线性化方法**

在当前参考点z₀处对曲率约束进行泰勒展开:

```
κ(z) ≈ κ(z₀) + ∇κ(z₀)·(z - z₀)
```

其中:
- z: 决策变量(控制点)
- z₀: 当前参考点
- ∇κ(z₀): 曲率在z₀处的梯度

**线性化曲率约束**

将曲率约束线性化:

```
原约束: |κ(z)| ≤ κ_max
线性化: |κ(z₀) + ∇κ(z₀)·Δz| ≤ κ_max
```

展开为两个线性约束:

```
κ(z₀) + ∇κ(z₀)·Δz ≤ κ_max    (上界约束)
κ(z₀) + ∇κ(z₀)·Δz ≥ -κ_max   (下界约束)
```

#### 2.3.4 曲率梯度计算

**梯度公式**

曲率对速度和加速度的偏导数:

```
∂κ/∂ẋ = (3·ẋ·ẏ·(ẏ·ẍ - ẋ·ÿ)) / (ẋ² + ẏ²)^(5/2)
∂κ/∂ẏ = (3·ẋ·ẏ·(ẋ·ÿ - ẏ·ẍ)) / (ẋ² + ẏ²)^(5/2)
∂κ/∂ẍ = -ẏ / (ẋ² + ẏ²)^(3/2)
∂κ/∂ÿ = ẋ / (ẋ² + ẏ²)^(3/2)
```

**梯度计算步骤**

1. **计算当前速度和加速度**
   ```
   ẋ, ẏ = trajectory.EvalDerivative(t, 1)
   ẍ, ÿ = trajectory.EvalDerivative(t, 2)
   ```

2. **计算当前曲率κ₀**
   ```
   κ₀ = (ẋ·ÿ - ẏ·ẍ) / (ẋ² + ẏ²)^1.5
   ```

3. **计算曲率梯度∇κ**
   ```
   ∂κ/∂ẋ = (3·ẋ·ẏ·(ẏ·ẍ - ẋ·ÿ)) / (ẋ² + ẏ²)^2.5
   ∂κ/∂ẏ = (3·ẋ·ẏ·(ẋ·ÿ - ẏ·ẍ)) / (ẋ² + ẏ²)^2.5
   ∂κ/∂ẍ = -ẏ / (ẋ² + ẏ²)^0.5
   ∂κ/∂ÿ = ẋ / (ẋ² + ẏ²)^0.5
   ```

4. **构建线性约束**
   ```
   linear_expr = ∂κ/∂ẋ·Δẋ + ∂κ/∂ẏ·Δẏ + ∂κ/∂ẍ·Δẍ + ∂κ/∂ÿ·Δÿ
   ```

#### 2.3.5 信任区域约束

**动机**

线性化仅在参考点附近有效,远离参考点时误差较大。信任区域约束限制优化变量的变化范围,确保线性化近似有效。

**约束形式**

```
‖Δz‖₂ ≤ δ
```

其中:
- Δz = z - z₀: 变量变化量
- δ: 信任区域半径

**Lorentz锥实现**

信任区域约束可以表示为Lorentz锥约束:

```
[δ, Δz₁, Δz₂, ..., Δzₙ] ∈ Lorentz锥
```

等价于:

```
δ ≥ √(Δz₁² + Δz₂² + ... + Δzₙ²)
```

#### 2.3.6 SCP迭代流程

**算法流程**

```
输入: 初始轨迹z₀, 最大曲率κ_max, 初始信任区域δ₀
输出: 满足曲率约束的轨迹z*

1. 初始化:
   z ← z₀
   δ ← δ₀
   iteration ← 0

2. While iteration < max_iterations:
   
   2.1 计算曲率违反量:
       violation = max(|κ(z)|) - κ_max
   
   2.2 检查收敛:
       If violation < tolerance:
           返回 z (收敛)
   
   2.3 线性化曲率约束:
       在z处计算κ₀和∇κ
       构建线性约束: |κ₀ + ∇κ·Δz| ≤ κ_max
   
   2.4 构建信任区域约束:
       ‖Δz‖₂ ≤ δ
   
   2.5 求解凸优化问题:
       z_new ← SolveConvexProblem()
   
   2.6 计算改进量:
       improvement = violation - max(|κ(z_new)|)
   
   2.7 更新信任区域:
       If improvement > 0:
           z ← z_new
           δ ← δ · expand_factor  (扩大信任区域)
       Else:
           δ ← δ · shrink_factor  (缩小信任区域)
   
   2.8 iteration ← iteration + 1

3. 返回 z (达到最大迭代次数)
```

**动态信任区域调整**

信任区域半径根据优化效果动态调整:

| 情况 | 改进量 | 信任区域调整 | 说明 |
|------|--------|-------------|------|
| 改进有效 | > 0 | 扩大(δ ← δ·1.5) | 线性化近似好,可以大胆优化 |
| 改进无效 | ≤ 0 | 缩小(δ ← δ·0.5) | 线性化近似差,需要保守优化 |

**收敛判据**

SCP的收敛判据:

1. **曲率违反量**: max(|κ(z)|) - κ_max < tolerance
2. **信任区域过小**: δ < δ_min
3. **达到最大迭代次数**: iteration ≥ max_iterations

#### 2.3.7 SCP参数配置

| 参数 | 默认值 | 说明 | 调整建议 |
|------|--------|------|---------|
| max_iterations | 50 | 最大迭代次数 | 复杂场景可增加到100 |
| convergence_tolerance | 1e-3 | 收敛容差 | 高精度需求可减小到1e-4 |
| initial_trust_region_radius | 1.0 | 初始信任区域半径 | 根据问题规模调整 |
| min_trust_region_radius | 1e-6 | 最小信任区域半径 | 通常无需调整 |
| trust_region_expand_factor | 1.5 | 信任区域扩大因子 | 可在1.2-2.0之间调整 |
| trust_region_shrink_factor | 0.5 | 信任区域缩小因子 | 可在0.3-0.7之间调整 |

---

### 2.4 工作空间约束

工作空间约束确保轨迹始终在可行区域内,避免碰撞障碍物。

#### 2.4.1 约束形式

**数学形式**

```
P_i ∈ Region,  i = 0, 1, ..., n
```

其中:
- P_i: 第i个控制点
- Region: 可行区域(凸集)

**凸集表示**

规划器支持多种凸集表示:

| 凸集类型 | 表示形式 | 适用场景 |
|---------|---------|---------|
| HPolyhedron | A·x ≤ b | 多面体区域,最常用 |
| Hyperellipsoid | (x-c)ᵀAᵀA(x-c) ≤ 1 | 椭球区域 |
| Point | x = x₀ | 单点区域 |

#### 2.4.2 实现方式

**控制点约束**

贝塞尔曲线的凸包性质:

```
如果所有控制点都在凸集C内,则整条曲线都在C内
```

因此,只需约束控制点在工作空间内,即可保证整条轨迹在工作空间内。

**GCS顶点构建**

每个GCS顶点对应一个区域,顶点的凸集为:

```
Vertex_Set = Region^(order+1) × TimeScalingSet
```

其中:
- Region^(order+1): (order+1)个空间控制点的约束
- TimeScalingSet: 时间控制点的约束

---

### 2.5 连续性约束

连续性约束确保轨迹在连接点处平滑,满足指定的连续性阶数。

#### 2.5.1 连续性阶数

| 阶数 | 名称 | 物理意义 | 约束内容 |
|------|------|---------|---------|
| C0 | 位置连续 | 轨迹不断裂 | 位置相等 |
| C1 | 速度连续 | 速度不突变 | 位置、速度相等 |
| C2 | 加速度连续 | 加速度不突变 | 位置、速度、加速度相等 |
| C3 | 加加速度连续 | 加加速度不突变 | 位置、速度、加速度、加加速度相等 |

#### 2.5.2 约束形式

**数学形式**

```
d^k r_v/ds^k (s=0) = d^k r_u/ds^k (s=1),  k = 0, 1, ..., continuity
```

其中:
- r_u: 前一段轨迹
- r_v: 后一段轨迹
- k: 导数阶数

**线性等式约束**

连续性约束是线性等式约束:

```
A·x = b
```

其中:
- x: 相邻两段轨迹的控制点
- A, b: 由贝塞尔基函数导数确定的系数

#### 2.5.3 配置建议

| 应用场景 | 推荐连续性 | 说明 |
|---------|-----------|------|
| 简单路径规划 | C0 | 仅需位置连续 |
| 平滑轨迹规划 | C1 | 速度连续,适合移动机器人 |
| 高质量轨迹 | C2 | 加速度连续,适合自动驾驶 |
| 超平滑轨迹 | C3 | 加加速度连续,适合高精度场景 |

---

### 2.6 时间缩放约束

时间缩放约束确保时间参数化单调递增,即轨迹只能向前,不能倒退。

#### 2.6.1 约束形式

**数学形式**

```
dh/ds ≥ hdot_min > 0
```

其中:
- h(s): 时间轨迹(关于参数s)
- hdot_min: 时间导数的最小值

**物理意义**

时间导数dh/ds表示时间随参数s的变化率:

- dh/ds > 0: 时间递增(向前运动)
- dh/ds = 0: 时间停滞(静止)
- dh/ds < 0: 时间递减(向后运动,不允许)

#### 2.6.2 实现方式

**控制点约束**

对时间轨迹的控制点施加约束:

```
h_{i+1} - h_i ≥ hdot_min,  i = 0, 1, ..., n-1
```

**线性不等式约束**

时间缩放约束是线性不等式约束:

```
A·h ≤ b
```

其中:
- h: 时间控制点
- A, b: 约束系数

#### 2.6.3 参数选择

| 参数 | 典型值 | 说明 |
|------|--------|------|
| hdot_min | 1e-6 | 时间导数最小值,防止速度突变 |

---

## 3. 成本设计

成本设计决定了轨迹的优化目标,影响轨迹的质量和特性。AckermannGCS规划器提供多种成本项,可根据应用场景灵活配置。

### 3.1 时间成本

时间成本最小化轨迹的总时间,使车辆尽快到达目标。

#### 3.1.1 数学形式

```
J_time = w_time · T_total = w_time · (h(1) - h(0))
```

其中:
- w_time: 时间成本权重
- T_total: 轨迹总时间
- h(1): 终点时间
- h(0): 起点时间

#### 3.1.2 成本类型

时间成本是**线性成本**:

```
J = cᵀ·x + d
```

其中:
- c: 成本系数向量
- x: 决策变量(时间控制点)
- d: 常数项

线性成本是凸成本,保持优化问题的凸性。

#### 3.1.3 物理意义

最小化时间成本相当于:

- 使车辆以最快速度到达目标
- 优化时间分配,减少不必要的停留
- 提高任务完成效率

#### 3.1.4 权重配置

| 应用场景 | 推荐权重 | 说明 |
|---------|---------|------|
| 时间最优 | 1.0-10.0 | 优先最小化时间 |
| 平衡规划 | 0.1-1.0 | 平衡时间和其他目标 |
| 舒适规划 | 0.01-0.1 | 时间次要,舒适优先 |

---

### 3.2 路径长度成本

路径长度成本最小化轨迹的空间长度,使轨迹更短。

#### 3.2.1 数学形式

**精确形式**

```
J_length = w_length · ∫‖dr/ds‖ds
```

其中:
- w_length: 路径长度成本权重
- r(s): 空间轨迹(关于参数s)

**近似形式**

规划器使用L2范数成本近似:

```
J_length ≈ w_length · Σ ‖dr/ds‖_i
```

其中求和遍历所有控制点。

#### 3.2.2 成本类型

路径长度成本是**L2范数成本**:

```
J = ‖A·x + b‖₂
```

L2范数成本是凸成本,保持优化问题的凸性。

#### 3.2.3 物理意义

最小化路径长度成本相当于:

- 使轨迹更短,减少行驶距离
- 避免不必要的绕路
- 提高路径效率

#### 3.2.4 与时间成本的区别

| 成本类型 | 优化目标 | 适用场景 |
|---------|---------|---------|
| 时间成本 | 最小化时间 | 时间敏感任务 |
| 路径长度成本 | 最小化距离 | 距离敏感任务 |
| 两者结合 | 平衡时间和距离 | 综合优化 |

---

### 3.3 能量成本

能量成本最小化轨迹的动能积分,使运动更节能。

#### 3.3.1 数学形式

**精确形式**

```
J_energy = w_energy · ∫‖dr/dt‖²dt
```

其中:
- w_energy: 能量成本权重
- dr/dt: 速度向量

**参数化形式**

使用参数s表示:

```
J_energy = w_energy · ∫‖r'(s)‖² / h'(s) ds
```

其中:
- r'(s) = dr/ds: 空间轨迹对参数s的导数
- h'(s) = dh/ds: 时间轨迹对参数s的导数

#### 3.3.2 成本类型

能量成本是**透视二次成本**:

```
J = ‖A·x‖² / (b·x)
```

透视二次成本是凸成本,保持优化问题的凸性。

#### 3.3.3 物理意义

最小化能量成本相当于:

- 减少速度波动,保持匀速运动
- 降低能耗,提高能源效率
- 减少电机负担,延长设备寿命

#### 3.3.4 与路径长度成本的区别

| 成本类型 | 数学形式 | 优化目标 |
|---------|---------|---------|
| 路径长度成本 | ∫‖v‖dt | 最小化距离 |
| 能量成本 | ∫‖v‖²dt | 最小化能量 |

路径长度成本优化速度的一次方,能量成本优化速度的二次方。能量成本对速度变化更敏感,更倾向于匀速运动。

---

### 3.4 曲率惩罚成本

曲率惩罚成本惩罚轨迹的曲率,使轨迹更平滑。

#### 3.4.1 三种成本形式

规划器提供三种曲率惩罚成本:

| 成本类型 | 数学形式 | 物理意义 | 适用场景 |
|---------|---------|---------|---------|
| **曲率平方积分** | J = ∫κ²(s)ds | 平滑度 | 通用平滑 |
| **曲率导数积分** | J = ∫(dκ/ds)²ds | 曲率变化率 | 超平滑轨迹 |
| **曲率峰值惩罚** | J = max(κ²(s)) | 最大曲率 | 限制急转弯 |

#### 3.4.2 曲率平方积分

**数学形式**

```
J_curvature = w_curvature · ∫κ²(s)ds
```

**物理意义**

- 惩罚曲率大的区域,减少急转弯
- 使轨迹更平滑,提高舒适性
- 减少转向系统负担

**数值积分**

使用高斯-勒让德积分计算:

```
∫κ²(s)ds ≈ Σ w_i · κ²(s_i) · ‖r'(s_i)‖
```

其中:
- s_i: 积分采样点
- w_i: 积分权重
- ‖r'(s_i)‖: 速度模长(雅可比)

#### 3.4.3 曲率导数积分

**数学形式**

```
J_curvature_deriv = w_deriv · ∫(dκ/ds)²ds
```

**物理意义**

- 惩罚曲率变化率,使曲率更均匀
- 避免曲率突变,提高超平滑性
- 适合高精度场景

**数值计算**

使用有限差分计算曲率导数:

```
dκ/ds ≈ (κ_{i+1} - κ_{i-1}) / (2·Δs)
```

#### 3.4.4 曲率峰值惩罚

**数学形式**

```
J_curvature_peak = w_peak · max(κ²(s))
```

**物理意义**

- 惩罚最大曲率,限制急转弯
- 确保所有点的曲率都不超过阈值
- 适合安全性要求高的场景

**数值计算**

在采样点中寻找最大曲率:

```
max(κ²(s)) ≈ max(κ²(s_i)),  i = 1, 2, ..., n
```

#### 3.4.5 凸松弛方法

**动机**

曲率成本是非凸的,直接添加会破坏优化问题的凸性。凸松弛方法通过近似保持凸性。

**松弛形式**

使用二阶导数平方积分近似曲率平方积分:

```
J_relaxed = ∫‖r''(s)‖²ds
```

**近似原理**

曲率的近似:

```
κ ≈ ‖r''(s)‖ / ‖r'(s)‖²
```

当速度近似恒定时(‖r'(s)‖ ≈ const):

```
κ² ∝ ‖r''(s)‖²
```

**优点**

- r''(s)是控制点的线性函数
- ‖r''(s)‖²是凸函数
- 保持优化问题的凸性

**缺点**

- 仅在速度近似恒定时准确
- 对速度变化大的轨迹近似误差大

#### 3.4.6 数值积分方法

规划器支持三种数值积分方法:

| 方法 | 精度 | 速度 | 推荐度 |
|------|------|------|--------|
| **高斯-勒让德** | 高 | 中 | ⭐⭐⭐⭐⭐ |
| **梯形** | 低 | 快 | ⭐⭐⭐ |
| **辛普森** | 中 | 中 | ⭐⭐⭐⭐ |

**高斯-勒让德积分**

在区间[0, 1]上的积分:

```
∫f(s)ds ≈ Σ w_i · f(s_i)
```

其中(s_i, w_i)是高斯-勒让德节点和权重。

**梯形积分**

```
∫f(s)ds ≈ (1/n) · [0.5·f(s_0) + f(s_1) + ... + f(s_{n-1}) + 0.5·f(s_n)]
```

**辛普森积分**

```
∫f(s)ds ≈ (h/3) · [f(s_0) + 4·f(s_1) + 2·f(s_2) + 4·f(s_3) + ... + f(s_n)]
```

其中h = 1/n。

---

### 3.5 导数正则化成本

导数正则化成本惩罚高阶导数,使轨迹更平滑。

#### 3.5.1 数学形式

```
J_reg = w_r·∫‖d^k r/ds^k‖²ds + w_h·∫‖d^k h/ds^k‖²ds
```

其中:
- w_r: 空间轨迹导数的正则化权重
- w_h: 时间轨迹导数的正则化权重
- k: 导数阶数

#### 3.5.2 成本类型

导数正则化成本是**二次成本**:

```
J = xᵀ·Q·x + cᵀ·x + d
```

其中Q是正半定矩阵,因此二次成本是凸成本。

#### 3.5.3 物理意义

| 导数阶数 | 物理意义 | 作用 |
|---------|---------|------|
| k=2 | 加速度 | 减少加速度波动,提高舒适性 |
| k=3 | 加加速度 | 减少加加速度波动,提高超舒适性 |
| k=4 | 四阶导数 | 减少四阶导数波动,提高超超舒适性 |

#### 3.5.4 作用

- 减少轨迹振荡,提高平滑度
- 改善数值稳定性,避免病态问题
- 提高求解效率,加速收敛

## 4. 约束与成本的协同设计

约束和成本需要协同设计,以实现高质量的轨迹规划。

### 4.1 凸性保证

| 组件 | 凸性 | 处理方法 | 说明 |
|------|------|---------|------|
| 航向角约束 | ✓ 凸 | 旋转矩阵法 | 线性等式约束 |
| 速度约束 | ✓ 凸 | Lorentz锥约束 | 二阶锥约束 |
| 工作空间约束 | ✓ 凸 | HPolyhedron | 多面体约束 |
| 连续性约束 | ✓ 凸 | 线性等式约束 | 仿射约束 |
| 时间缩放约束 | ✓ 凸 | 线性不等式约束 | 半空间约束 |
| 曲率约束 | ✗ 非凸 | SCP线性化 | 序列凸规划 |
| 时间成本 | ✓ 凸 | 线性成本 | 仿射成本 |
| 路径长度成本 | ✓ 凸 | L2范数成本 | 范数成本 |
| 能量成本 | ✓ 凸 | 透视二次成本 | 透视成本 |
| 曲率成本 | ✗ 非凸 | 凸松弛或SCP | 近似或迭代 |
| 导数正则化成本 | ✓ 凸 | 二次成本 | 二次形式 |

### 4.2 约束层次

约束按重要性分为三个层次:

| 层次 | 约束类型 | 处理方式 | 说明 |
|------|---------|---------|------|
| **硬约束** | 工作空间、连续性、时间缩放 | 必须满足 | 物理可行性 |
| **软约束** | 速度、航向角 | 尽量满足 | 质量要求 |
| **迭代约束** | 曲率 | SCP迭代 | 非凸处理 |

### 4.3 成本权重配置

成本权重决定了各成本项的相对重要性:

```
J_total = w_time·J_time + w_length·J_length + w_energy·J_energy + w_curvature·J_curvature + w_reg·J_reg
```

**权重配置原则**

1. **归一化**: 将各成本项归一化到相似量级
2. **优先级**: 根据应用场景确定优先级
3. **平衡**: 避免某一成本项主导优化

**典型配置**

| 应用场景 | w_time | w_length | w_energy | w_curvature | w_reg |
|---------|--------|----------|----------|-------------|-------|
| 时间最优 | 10.0 | 0.1 | 0.01 | 0.1 | 0.01 |
| 距离最优 | 0.1 | 10.0 | 0.01 | 0.1 | 0.01 |
| 能量最优 | 0.1 | 0.1 | 10.0 | 0.1 | 0.01 |
| 平滑最优 | 0.1 | 0.1 | 0.1 | 10.0 | 0.1 |
| 平衡规划 | 1.0 | 1.0 | 1.0 | 1.0 | 0.1 |

## 5. 求解流程

### 5.1 整体流程

```
输入: 起点、终点、工作空间、车辆参数
输出: 满足约束的轨迹

1. 初始化AckermannBezierGCS
   ├── 构建GCS图
   ├── 添加工作空间约束
   ├── 添加连续性约束
   └── 添加时间缩放约束

2. 添加起终点约束
   ├── addSourceTarget() [位置约束]
   └── _add_heading_constraint() [航向角约束]

3. 添加速度约束
   └── addScalarVelocityLimit() [SOCP]

4. 添加成本函数
   ├── addTimeCost()
   ├── addPathLengthCost()
   ├── addPathEnergyCost()
   └── CurvatureCostModule.add_curvature_cost_to_gcs()

5. SCP迭代求解
   ├── 求解初始轨迹(忽略曲率约束)
   └── While 未收敛:
       ├── 计算曲率违反量
       ├── 线性化曲率约束
       ├── 构建信任区域约束
       ├── 求解GCS优化问题
       └── 调整信任区域

6. 返回最终轨迹
```

### 5.2 GCS求解

GCS求解是凸优化求解,使用Drake的GCS算法:

```
输入: GCS图、约束、成本
输出: 最优轨迹

1. 凸松弛
   └── 将整数变量松弛为连续变量

2. 求解凸优化
   └── 使用MOSEK求解器

3. 舍入(Rounding)
   └── 将松弛解舍入为整数解

4. 提取轨迹
   └── 从GCS解中提取控制点
```

### 5.3 SCP迭代

SCP迭代处理非凸曲率约束:

```
输入: 初始轨迹、曲率约束
输出: 满足曲率约束的轨迹

1. 初始化
   ├── trajectory ← initial_trajectory
   ├── delta ← initial_trust_region_radius
   └── iteration ← 0

2. While iteration < max_iterations:
   
   2.1 计算曲率违反量
       violation ← compute_curvature_violation(trajectory)
   
   2.2 检查收敛
       If violation < tolerance:
           Return trajectory
   
   2.3 线性化曲率约束
       linearized_constraints ← linearize_curvature(trajectory)
   
   2.4 构建信任区域约束
       trust_region ← build_trust_region(delta)
   
   2.5 求解GCS
       new_trajectory ← solve_gcs(linearized_constraints, trust_region)
   
   2.6 计算改进量
       improvement ← violation - compute_curvature_violation(new_trajectory)
   
   2.7 更新信任区域
       If improvement > 0:
           trajectory ← new_trajectory
           delta ← delta * expand_factor
       Else:
           delta ← delta * shrink_factor
   
   2.8 iteration ← iteration + 1

3. Return trajectory
```

## 6. 关键设计特点

### 6.1 约束设计特点

#### 6.1.1 航向角约束创新

**旋转矩阵法**

- 使用叉积形式避免奇异
- 数值稳定,适用于所有角度
- 凸性约束,可嵌入GCS

**方向约束**

- 使用点积形式确保朝向一致
- 解决航向角约束的方向歧义问题
- 与叉积约束结合,完全确定方向

**多控制点约束**

- 约束多个控制点对,提高轨迹质量
- 减少轨迹振荡
- 增强鲁棒性

#### 6.1.2 速度约束优化

**标量约束**

- 比矢量约束更精确,可行域大27%
- 使用SOCP保持凸性
- 更符合阿克曼车辆物理模型

**Lorentz锥实现**

- 高效的二阶锥约束实现
- 可使用MOSEK等SOCP求解器
- 求解效率高

#### 6.1.3 曲率约束处理

**SCP迭代**

- 处理非凸约束的标准方法
- 通过线性化将非凸问题转化为凸问题
- 迭代逼近最优解

**动态信任区域**

- 根据优化效果动态调整信任区域
- 提高收敛速度
- 增强鲁棒性

**多层次提前终止**

- 检查曲率违反量、改进率、信任区域等多个指标
- 及时终止无效迭代
- 提高求解效率

### 6.2 成本设计特点

#### 6.2.1 多样化成本项

**五种成本类型**

- 时间成本: 最小化时间
- 路径长度成本: 最小化距离
- 能量成本: 最小化能量
- 曲率惩罚成本: 最小化曲率
- 导数正则化成本: 最小化高阶导数

**灵活配置**

- 支持权重配置
- 可根据应用场景调整
- 可扩展性强

#### 6.2.2 数值积分精度

**三种积分方法**

- 高斯-勒让德积分: 精度最高
- 梯形积分: 速度最快
- 辛普森积分: 中等精度

**可配置采样点数**

- 根据精度需求调整
- 平衡精度和效率

#### 6.2.3 凸松弛技术

**曲率成本凸松弛**

- 使用二阶导数平方积分近似曲率平方积分
- 保持优化问题凸性
- 提高求解效率

**适用条件**

- 速度近似恒定时准确
- 对速度变化大的轨迹近似误差大

### 6.3 数值稳定性

#### 6.3.1 旋转矩阵法

- 避免tan函数在θ ≈ ±π/2时的奇异
- 使用sin和cos函数,数值稳定
- 适用于所有角度

#### 6.3.2 信任区域

- 限制优化变量的变化范围
- 确保线性化近似有效
- 防止优化发散

#### 6.3.3 数值容差

- 避免除零错误
- 处理数值舍入误差
- 提高鲁棒性

## 8. 性能优化建议

### 8.1 求解效率优化

#### 8.1.1 减少约束数量

- 使用必要的约束,避免冗余约束
- 合并相似约束
- 使用稀疏约束表示

#### 8.1.2 调整SCP参数

- 增大收敛容差,减少迭代次数
- 调整信任区域参数,加速收敛
- 使用提前终止,避免无效迭代

#### 8.1.3 并行化

- 并行计算曲率线性化
- 并行计算约束违反量
- 使用多线程求解

### 8.2 解的质量优化

#### 8.2.1 提高贝塞尔阶数

- 增加贝塞尔曲线阶数,提高灵活性
- 阶数越高,轨迹越平滑
- 但求解时间增加

#### 8.2.2 调整成本权重

- 根据应用场景调整权重
- 平衡各成本项的重要性
- 避免某一成本项主导优化

#### 8.2.3 增加约束强度

- 增加多控制点约束数量
- 提高连续性阶数
- 减小约束容差

### 8.3 数值稳定性优化

#### 8.3.1 调整数值容差

- 增大数值容差,避免除零错误
- 但可能降低精度
- 需要平衡稳定性和精度

#### 8.3.2 使用稳定的约束方法

- 使用旋转矩阵法处理航向角约束
- 使用标量约束处理速度约束
- 避免使用数值不稳定的方法

#### 8.3.3 调整信任区域

- 增大初始信任区域半径
- 减小信任区域缩小因子
- 避免信任区域过小导致收敛困难
