# 参数管理方案

本文档说明当前项目的统一参数入口。目标是把常见调参从 Python 源码迁移到 YAML，同时保留现有 dataclass 的类型校验和运行时对象。

## 配置入口

统一加载函数：

```python
from config.project import load_project_config

project_config = load_project_config(
    "config/experiments/default.yaml",
    overrides=["ackermann.vehicle.max_velocity=8.0"],
)
```

配置优先级固定为：

```text
dataclass 默认值 < base YAML/extends < experiment YAML < CLI --set
```

默认实验文件：

- `config/experiments/default.yaml`
- `config/experiments/curvature_direction_cone.yaml`
- `config/scenarios/default.yaml`

## 目录组织

当前 `config/` 按职责分为：

- `project/`：统一 YAML schema、加载器、CLI override 和运行时适配器。
- `experiments/`：可直接传给 `--config` 的实验配置。
- `scenarios/`：场景库，被实验 YAML 通过 `extends` 继承。
- `environments/`：Conda 环境文件，例如 `iris_env.yaml`。
- `iris/`、`gcs/`、`solver/`、`planner/`、`visualization/`：仍被算法模块直接 import 的运行时 dataclass/预设。

## YAML 结构

顶层配置按职责分组：

- `scenario` 和 `scenarios`：当前场景、地图大小、起终点、走廊宽度。
- `astar`：Hybrid A* 搜索半径、分辨率、迭代上限和容差。
- `corridor`：走廊宽度、平滑窗口和边界裕度。
- `iris`：IRIS 开关、模式和种子/采样参数。
- `gcs`：Bezier 阶数、连续性、成本权重、rounding 和求解器参数。
- `ackermann`：车辆、Bezier、轨迹约束、曲率模式和 direction cone 预设。
- `visualization`：采样数、2D/3D 显示、控制点和保存参数。
- `batch`：批量测试场景、次数和静默输出。

YAML 中角度统一使用 `*_deg`，加载后由适配器转换为弧度。例如：

```yaml
ackermann:
  vehicle:
    max_steering_angle_deg: 85.0

scenarios:
  gentle_turn:
    goal:
      heading_deg: 30.0
```

## 常用命令

主规划脚本：

```bash
python3 scripts/hybrid_astar_gcs_planner.py basic ackermann_gcs \
  --config config/experiments/curvature_direction_cone.yaml
```

3D 可视化：

```bash
python3 scripts/visualize_3d_trajectory.py basic \
  --config config/experiments/default.yaml \
  --set visualization.elev=35.0
```

批量测试：

```bash
python3 scripts/batch_test_curvature_constraint.py \
  --config config/experiments/curvature_direction_cone.yaml \
  --set batch.num_runs=3
```

导出最终生效配置：

```bash
python3 scripts/hybrid_astar_gcs_planner.py --dump-config output/resolved_config.yaml
```

## 代码适配器

`ProjectConfig` 提供现有对象的转换方法：

- `vehicle_params()` -> `VehicleParams`
- `bezier_config()` -> `BezierConfig`
- `trajectory_constraints(workspace_regions)` -> `TrajectoryConstraints`
- `planner_config(scenario_name)` -> `PlannerConfig`
- `astar_planner_config()` -> Hybrid A* `PlannerConfig`
- `visualization_config()` -> `VisualizationConfig`
- `cost_weights()` -> `dict`

新增脚本或测试应优先接收 `ProjectConfig`，再调用这些适配器生成模块需要的运行时对象。

## 注意事项

- `config/scenarios/default.yaml` 当前只迁移场景元数据；障碍物构造逻辑仍在 `create_test_map()` 中。
- `scripts/hybrid_astar_gcs_planner.py` 不再导出旧的 `SCENARIO_CONFIGS`、`DEFAULT_VEHICLE_PARAMS`、`CURVATURE_CONSTRAINT_MODE` 等兼容别名；新增代码应通过 `load_project_config()` 获取参数。
- 详细参数含义和建议取值见 `docs/YAML参数配置参考.md`。
- 修改完整 IRIS/GCS/Ackermann 链路后，仍需在 Ubuntu `iris-py3.12` 环境中验证 Drake 和求解器行为。
